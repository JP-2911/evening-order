# Evening Order Product Language Compiler

Purpose:
Unify and optimize the language system of the product based on the Evening Order Language Manual.

Core principles:
- Warm
- Understanding
- Non-judgmental
- Invitation instead of command
- Human language

Vocabulary rules:
- "上床时间" → "躺下时间"
- "入睡时间" → do not use
- Use:
  - 目标躺下时间
  - 实际躺下时间
  - 躺下节律

Feedback structure:
Observation → Meaning → Suggestion → Encouragement

Encouragement levels:
1. Close to target
2. Clearly improving
3. Goal reached
4. Rhythm becoming stable

Do not use:
- 拖延
- 自制力差
- 失败
- 沉迷
- 不够努力

Suggestion style:
Use invitation-based phrasing, such as:
- 如果愿意，可以试试……
- 也许可以尝试……
- 可以考虑……

Do not modify:
- analysis.ts logic
- patternKey
- storage structure
- data schema

Only update:
- copywriting.ts
- user-facing product language
- feedback wording
- report wording
